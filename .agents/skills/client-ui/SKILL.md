## 客户端开发指南

## 基本

1. 强制使用vue3 setup api 配合 ts语法，样式使用unocss配合vue3 style标签。
  - pages和modules层尽量使用unocss，components层可以适当使用style标签来编写组件样式，但要注意保持组件的样式简洁和可维护。
2. 禁止使用computed和deepwatch，禁止使用watchEffect
3. 涉及到复杂对象时，尽量使用 浅层watch配合shallowRef和traiggerRef 来实现响应式，禁止使用深层watch
4. 浅层使用ref，禁止使用reactive，禁止使用shallowReactive
5. 禁止复杂hook，非必要不抽象hook
6. 生成的代码要简洁！不要有太多注释，使用有意义的变量名和函数名来表达代码的意图，避免冗余的代码和不必要的抽象
7. 组件设计要简单！每个组件只负责一个功能，避免过于复杂的组件，组件之间要有清晰的层次关系，避免过多的嵌套和循环引用
8. 代码的类型要完备！每个变量、函数参数和返回值都要有明确的类型定义，避免使用any类型，尽量使用接口和类型别名来定义复杂的类型结构
9. vue文件不允许导出任何变量或函数，禁止使用export default以外的导出方式，不允许导出类型。
10. 所有文件都应该是小写命名，横线分隔，禁止使用驼峰命名或下划线命名，禁止在文件名中使用大写字母。
11. 所有代码尽量以扁平化函数式编程的方式来实现，禁止使用类组件和面向对象编程的方式来实现组件和页面，禁止使用复杂的设计模式来实现组件和页面。
12. 简单的代码如果跟vueuse有关，请参考 [vueuse-functions](../vueuse-functions/SKILL.md) 中的相关函数。

## 项目结构

- components: 存放可复用的组件，如按钮、表单项等
  - [组件名称]: 每个组件一个目录，目录名为组件名称，目录下可以有index.vue和style.css等相关文件
	  - index.vue: 组件的实现文件，包含组件的模板、脚本和样式
	  - style.css: 组件的样式文件，包含组件的样式定义
		- types.ts: 组件的类型定义文件，包含组件的props、emits等类型定义
		- index.ts: 组件的导出文件，导出组件的类型定义和组件本身，禁止在index.vue中直接导出组件
- modules: 存放与业务相关的组件，这些组件通常会组合components中的组件来实现具体的业务功能，如用户管理模块、权限管理模块等，每个模块一个目录，目录名为模块名称，目录下可以有index.vue和components子目录
- pages: 存放页面级组件，如登录页、首页等, 每个页面一个目录，目录名为页面名称，目录下可以有index.vue和components子目录
- store: 存放全局状态管理相关的代码，只使用 @vue/reactivity 来实现状态管理，不使用vuex或pinia等第三方库
- api: 存放与后端接口交互相关的代码，如api请求函数、接口定义等
- utils: 存放工具函数，如日期格式化、数据处理等
- types: 存放类型定义，如接口定义、类型别名等，这里只放全局使用的类型定义，局部使用的类型定义应该放在对应的组件文件中
- layouts: 存放布局组件，如公共的页面布局组件等
  - full.vue: 全屏布局组件，适用于登录页等不需要公共布局的页面
	- basic.vue: 基础布局组件，适用于需要公共布局的页面
	- landing.vue: 落地页布局组件，适用于落地页等需要特殊布局的页面
